## 1. 我们要解决什么问题？
有一张格子地图，有些格子是空地、有些是墙。给定起点和终点，要找一条 **最短的、不穿墙的** 路。这就是路径规划。

## 2. 先从最笨但能懂的方法说起：水波扩散（BFS）
想象从起点滴一滴水，水一圈一圈往外扩。第一次扩到终点时，走过的圈数就是最短距离。这就是广度优先搜索（BFS）。它能找到最短路，但 **没有方向感**——明明终点在右边，它还往左边扩一大片，浪费。

Dijkstra 是这个思路的「带路费」版本：每走一步可能代价不同，它总是优先扩展「当前累计代价最小」的格子。每步代价都一样时，它和 BFS 效果一样。

## 3. 关键升级：给它装一个「指南针」——启发式 h
我们希望搜索 **朝着终点方向** 优先走，少做无用功。于是给每个格子估一个「离终点还有多远」的值，叫启发式 h。

本项目用 **曼哈顿距离**：横向格数差 + 纵向格数差。比如当前在 (2,1)、终点在 (2,5)，h = |2-2| + |1-5| = 4。机器人只能上下左右走，这个估计恰好合理。

> ⚠️ 一个重要规则：**h 不能高估真实距离**（专业叫「可采纳性」）。曼哈顿距离永远 ≤ 真实要走的步数，所以安全。如果你乱估、估大了，A\* 可能贪快而错过真正的最短路。

## 4. A\* 的心脏：f = g + h
A\* 给每个待探索的格子算一个综合分数：
```
f = g + h
  g：从起点到这个格子，已经走了多远（真实、确定）
  h：从这个格子到终点，估计还要走多远（猜的）
  f：优先级，越小越优先探索
```
每一步，A\* 都从「还没探索的格子」里挑 **f 最小** 的那个来扩展。
直觉：g 让它不忘记「已经付出的代价」（保证最短），h 让它「朝目标使劲」（保证快）。
**A\* = Dijkstra 的稳 + 贪婪的快。**

## 5. 小例子手推（5×5，四方向，无墙）
起点 S=(0,0)，终点 G=(0,4)：
- 起点 (0,0)：g=0，h=|0-0|+|0-4|=4，f=4
- 邻居 (0,1)：g=1，h=3，f=4 ；邻居 (1,0)：g=1，h=5，f=6
- 两个邻居里 (0,1) 的 f=4 更小，于是优先扩展 (0,1) —— 它正好朝终点方向！
- 继续：(0,2) f=2+2=4 → (0,3) f=3+1=4 → (0,4) f=4+0=4 到达终点。

全程几乎一条直线冲过去，没怎么探索上下方的格子，这就是启发式的威力。
（换成 Dijkstra，h 全当 0，它会把四周均匀探一遍才到，`Expanded` 数字更大。你在程序里按 `2` 切到 Dijkstra 就能亲眼看到。）

## 6. 为什么用优先队列（最小堆 heapq）？
A\* 每一步都要「取出当前 f 最小的格子」。每次都遍历找最小太慢。最小堆能在 O(log n) 内取最小、插入新格子，正好契合。代码里就是 `open_heap`，存的是 `(优先级, 计数器, 格子)`（计数器用于优先级相同时稳定排序、避免报错）。

## 7. 对照本项目真实代码（pathfinding.py）
搜索主循环里几个关键变量：
- `g_score[格子]`：起点到该格子的真实代价 g
- `heuristic(格子, 终点)`：估计代价 h（曼哈顿距离）
- `open_heap`：优先队列，按 f 排序
- `came_from[格子]`：记录这个格子从哪走来，最后回溯路径用
- `closed`：已处理完的格子，避免重复

**三种算法只差这一行**（全篇最该记住的）：
```python
if algorithm == "dijkstra":
    priority = tentative_g                          # 只用 g
elif algorithm == "greedy":
    priority = heuristic(nxt, goal)                 # 只用 h
else:  # astar
    priority = tentative_g + heuristic(nxt, goal)   # f = g + h
```
看懂这三行，你就同时懂了三个算法。

## 8. 三种算法对比表
| 算法 | 优先级 | 用启发式? | 保证最短? | 快慢 / 探索范围 |
|---|---|---|---|---|
| Dijkstra | g | 否 | 是 | 慢，均匀铺开 |
| 贪婪 Greedy | h | 是 | 否 | 快，但可能绕远 |
| **A\*** | g+h | 是 | 是 | 快且准 ✅ |

## 9. 复杂度与边界情况
- **时间**：最坏约 O(E log N)（E 边数、N 格子数），每个格子进出堆、每次堆操作 log N。
- **空间**：O(N)，存 g 值、来路、堆。
- **找不到路**：当起点和终点被墙完全隔开，优先队列会被取空都没到终点，程序返回 `found=False`（对应单元测试的「无解」用例）。

## 10. 三个最可能被追问的点 —— 标准答案
1. **A\* 怎么工作？** 给每个格子算 f=g+h，每次扩展 f 最小的，既看已走代价又看离终点估计，所以又快又能找到最短路。
2. **和 Dijkstra 区别？** Dijkstra 没有 h（相当于 h=0），均匀扩展、慢；A\* 有 h 引导方向、探索更少。
3. **为什么启发式不能高估？** 高估会破坏最短性保证，可能让算法贪快错过最短路；曼哈顿距离不高估，所以安全。
