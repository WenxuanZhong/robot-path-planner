# 两天冲刺计划 — 招聘会前

> 目标：招聘会上你能 **打开它演示、清楚地讲 A\*、给出 GitHub 链接**。
> 每天按「上午 / 下午 / 晚上」推进，做完一项打个勾。

---

## ✅ Day 1：跑通 + 吃透 + 录演示

### 上午：把它跑起来，建立信心
- [ ] 打开 PowerShell，运行 `pip install -r requirements.txt`（已装过 pygame 可跳过）
- [ ] 运行 `python app.py`，照操作说明玩 10 分钟：画墙、设起终点、空格运行
- [ ] 运行 `python test_pathfinding.py`，看到 `All tests passed.`
- [ ] 按 `1`/`2`/`3` 切换三种算法各跑一次，观察顶部 `Expanded` 数字的差别

### 下午：吃透算法（最关键的一步）
- [ ] 打开 `算法讲解笔记.md`，从头读一遍
- [ ] 对照着读 `pathfinding.py`，重点搞懂 `plan()` 函数和三种 `priority` 那几行
- [ ] 拿一张纸，自己默写一遍：f=g+h、g 是什么、h 是什么
- [ ] 理解 `test_pathfinding.py` 里 4 个用例分别在测什么

### 晚上：做出漂亮的演示素材
- [ ] 用程序画 2-3 个好看的场景：① 一个简单迷宫 ② 一道大墙让它绕路 ③ 同一张图对比 A\* 和 Dijkstra 的 `Expanded`
- [ ] 按 `docs/HOW_TO_RECORD_DEMO.txt` 录一个 15-20 秒演示 GIF，存成 `docs/demo.gif`
- [ ] 截 2-3 张好看的图，留着发简历 / 给 HR 看

---

## ✅ Day 2：上线 + 练讲 + 备战

### 上午：传到 GitHub（让项目「可信、可查」）
- [ ] 注册 / 登录 GitHub，新建仓库 `robot-path-planner`（先不要勾选自动生成 README）
- [ ] 在项目文件夹里打开 PowerShell，依次运行：
```bash
git init
git add .
git commit -m "feat: A* mobile robot path planning visual simulator"
git branch -M main
git remote add origin https://github.com/你的用户名/robot-path-planner.git
git push -u origin main
```
- [ ] 打开仓库网页，确认 README 显示正常、`demo.gif` 能动
> 没装 git 的话：到 https://git-scm.com/download/win 下载安装，一路下一步即可。

### 下午：练讲（决定面试成败）
- [ ] 把 `面试问答.md` 的 A、B 两组读熟，尤其 **30 秒自我介绍**
- [ ] 对着手机录一遍自己讲项目（30 秒版 + 2 分钟版），回放找卡壳处
- [ ] 重点练 3 个问题：**A\* 怎么工作？和 Dijkstra 区别？为什么用启发式？**
- [ ] 练「这是不是 AI 写的」的坦诚回答（见问答 F 组）

### 晚上：备战招聘会
- [ ] 简历更新好（用 `简历写法.md`），打印几份
- [ ] 手机里存好：`demo.gif`、GitHub 链接、项目截图
- [ ] 笔记本确认能现场跑 `python app.py`，电充满
- [ ] 列「招聘会当天清单」：简历、笔记本、充电器、GitHub 链接、30 秒自我介绍

---

## 🆘 如果时间不够，优先保住这 3 件事
1. **能演示** —— 电脑 / 手机里有能跑的程序或 `demo.gif`
2. **会讲 A\*** —— 能说清 f=g+h、和 Dijkstra 的区别
3. **有链接** —— GitHub 上传好，简历上有链接

> 记住：一个你 **讲得清** 的小项目，远胜一个你讲不清的大项目。稳住，你已经有真东西了。
