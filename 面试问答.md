## A. 项目概述

**Q1. 30 秒介绍一下这个项目。**
我做了一个移动机器人路径规划的可视化仿真器，用 Python 写的。在一张可以用鼠标画障碍的格子地图上，机器人用 A\* 算法自动找出从起点到终点的最短、不撞墙的路径，并且用动画把「搜索过程、找到的最短路、机器人走过去」一步步演示出来。它还能一键切换 A\*、Dijkstra、贪婪三种算法做对比。

**Q2. 详细讲讲（2 分钟版）。**

**Q3. 为什么做这个项目？**
路径规划是移动机器人最核心的能力之一，扫地机器人、AGV、自动驾驶都离不开它。我想通过亲手实现 A\*（而不是调库）真正搞懂它的原理，同时做成可视化的，既帮自己理解，也方便演示。

**Q4. 最大的收获 / 难点是什么？**

## B. 算法原理（重点，必背）

**Q5. A\* 的核心思想是什么？**
A\* 是一种带「方向感」的最短路搜索。它给每个待探索的格子算一个优先级 f = g + h：g 是从起点到这个格子已经走的真实步数，h 是这个格子到终点的估计距离。每次都优先扩展 f 最小的格子，相当于「既考虑已经走了多远，又考虑离终点还有多远」，所以能又快又准地找到最短路。

**Q6. f、g、h 分别是什么？**
g = 起点到当前格子的真实代价（已确定的）；h = 当前格子到终点的估计代价（启发式，猜的）；f = g + h，是综合优先级，越小越优先探索。

**Q7. 什么是启发式（heuristic）？你用的什么？**
启发式就是「估计还有多远」的函数。我用的是曼哈顿距离，也就是横向格数差加纵向格数差，因为我的机器人只能上下左右走（四方向），曼哈顿距离正好匹配。

**Q8. 为什么曼哈顿距离能保证找到最短路？**
关键是它「从不高估」真实距离——这个性质叫**可采纳性（admissible）**。四方向网格里两点之间最少就得走曼哈顿距离那么多步，所以估计值永远 ≤ 真实值。只要启发式不高估，A\* 就保证找到最短路；如果高估了（比如把距离乘以 2），可能为了快而错过真正的最短路。

**Q9. A\* 和 Dijkstra 有什么区别？**
Dijkstra 没有启发式，相当于 h=0，只看已经走了多远，所以它向四面八方均匀扩展，最后也能找到最短路但探索的格子多、慢。A\* 多了 h，会朝着终点方向优先探索，探索的格子少很多。在我的程序里能直观看到：同一张图 A\* 的 `Expanded` 数字通常比 Dijkstra 小。

**Q10. A\* 和 BFS（广度优先）的区别？**
在每步代价都相等时，BFS 其实就是均匀扩散，能找最短路但和 Dijkstra 一样没方向感。A\* 有启发式引导，更高效。

**Q11. 为什么用优先队列（堆）？**
因为每一步都要取出当前 f 最小的格子来扩展。优先队列（我用 Python 的 `heapq` 最小堆）能在 O(log n) 内完成「插入」和「取最小」，比每次遍历找最小快得多。

**Q12. 时间 / 空间复杂度大概多少？**
最坏情况要探索大部分格子，设格子数 N、边数 E，时间约 O(E log N)，因为每个格子可能进出优先队列、每次堆操作是 log N。空间 O(N)，要存 g 值、来路、堆。实际有了好的启发式，A\* 探索的格子远少于 N。

**Q13. 贪婪最佳优先为什么快但不一定最短？**
贪婪只看 h（离终点的估计），完全不管已经走了多远，一头朝终点扎。没障碍时它很快，但遇到障碍容易绕远，因为它从不「反悔」之前的选择，所以不保证最短。

**Q14. 三种算法在你代码里到底差在哪？**
就差优先级一行：Dijkstra 是 `priority = g`，贪婪是 `priority = h`，A\* 是 `priority = g + h`。其它搜索框架完全一样。这也是我觉得最妙的地方——理解了这一点就同时理解了三个算法。

**Q15. `came_from` 和路径回溯是怎么做的？**
搜索时我用字典 `came_from` 记录「每个格子是从哪个格子最优地走过来的」。到达终点后，从终点顺着 `came_from` 一路回溯到起点，再反转，就得到完整路径（`reconstruct_path`）。

**Q16. 优先队列里出现重复 / 过期的格子怎么办？**
当我发现一条更短的路到某个格子时，会把它用更小的优先级再压进堆，于是堆里可能有同一格子的旧条目。我用一个 `closed` 集合，每次弹出时如果这个格子已处理过就跳过，旧条目就被自然忽略了。

## C. 工程实现

**Q17. 为什么把算法和界面分成两个文件？**
解耦。算法 `pathfinding.py` 不依赖 pygame，是纯函数，输入地图输出结果，这样能单独写单元测试，也方便以后换界面或复用。界面 `app.py` 只负责画图和交互。职责分离是基本工程习惯。

**Q18. `plan()` 返回了哪些东西？**
一个字典：`path`（最终路径）、`visited`（按顺序记录探索过的格子，用来播放搜索动画）、`cost`（路径长度）、`expanded`（一共探索了多少格子，衡量效率）、`found`（是否找到）。

**Q19. `visited` 为什么要按顺序存？**
为了做搜索动画。把探索的先后顺序记下来，界面里一帧一帧点亮，就能看到搜索「扩散」的过程，而不是一下子出结果。

**Q20. 你怎么保证算法是对的？**
写了单元测试 `test_pathfinding.py`，4 个用例：空地图直线最短应等于曼哈顿距离、中间有墙要绕路、目标被围死时正确返回无解、A\* 和 Dijkstra 算出的最短长度必须一致。全部通过。

**Q21. 地图用什么数据结构？**
二维列表 `grid[row][col]`，0 表示可通行、1 表示障碍。格子坐标统一用 `(行, 列)` 元组表示。

**Q22. 动画是怎么实现的？**
用一个状态机：按空格后先算出完整结果，然后进入「search」阶段逐帧点亮 `visited` 的格子；点完进入「path」阶段，让机器人（一个圆点）沿路径一格一格移动；走到终点进入「done」。每帧由 pygame 主循环以 60 FPS 驱动。

## D. 机器人 / 行业关联

**Q23. 路径规划在真实机器人里处于什么位置？**
它是机器人「导航」能力的核心。一般流程是：先用 SLAM 建图和定位，知道自己在哪、地图什么样；然后全局路径规划（像 A\*）算一条到目标的路线；再用局部规划 / 避障（像 DWA）实时躲开突然出现的障碍。我的项目对应其中的**全局规划**部分。

**Q24. 全局规划和局部规划有什么区别？**
全局规划基于已知地图，算一条整体最优路径，但不考虑突然冒出来的动态障碍；局部规划在机器人实际行进时结合传感器实时数据，在小范围内调整、避开动态障碍，同时尽量贴着全局路径走。

**Q25. 栅格地图是什么？真实机器人地图哪来的？**
栅格地图就是把空间切成一个个小格子，每格标记是否被占据，叫「占据栅格地图」，是最常用的地图表示之一。真实机器人一般用激光雷达或深度相机扫描环境，通过 SLAM 算法建出这种地图。

**Q26. 哪些产品用到路径规划？**
扫地机器人、仓储 AGV / 物流分拣、送餐机器人、自动驾驶、无人机航迹规划、游戏 NPC 寻路等。

## E. 扩展与不足

**Q27. 这个项目现在有什么局限？**
目前是四方向移动、静态障碍、离散栅格。机器人被简化成一个点，没考虑体积和转弯半径，也没有动态障碍。

**Q28. 怎么支持斜着走？**
把邻居从 4 方向扩展到 8 方向（我代码里已预留 `MOVES_8`），启发式相应改成对角距离或欧氏距离，否则会高估、破坏最短性保证。

**Q29. 如果障碍会动怎么办？**
需要动态重规划。简单做法是障碍变化时重新跑一次 A\*；更高效的是用 D\* Lite 这类增量重规划，只更新受影响的部分。

**Q30. 为什么有些场景要用 RRT 而不是 A\*？**
A\* 适合离散的低维网格。当空间连续、维度高（比如机械臂很多关节），格子会爆炸，这时用 RRT / RRT\* 这类随机采样方法更合适。
